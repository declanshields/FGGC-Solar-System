FGGC Solar System

Project Explanation

This page is dedicated to the Solar System project created for my course work in my second year module, Further Games and Graphics Concepts

Demonstration of the project.

If the video does not work, it can be found here.


This project was created during semester 1 of Further Games and Graphics Concepts(FGGC) at Staffordshire University.
It is a 3D model of the solar system and some of the planets moons, as well as the asteroid belt, which was created using a supplied DirectX 11 framework.


This model includes a camera focused at each planet, as well as the sun, alongside a 9th camera which is a free movement camera. This 9th camera can move in the x, y, and z axis, however it cannot rotate.
One thing I am proud of in this project is the way I recreated Saturn's rings. I tried loading an OBJ file for the planet, however none would work due to the coordinate scale used. I then attempted to create the rings in Blender, however this didn't work due to my lack of experience with the software. In the end, I took the same code I used to create the asteroid belt, and replicated it to create 3 of Saturn's rings.
This involved generating a random x and z coordinate within a circle with a specific radius, I then checked to see how far away from the center of the circle it was, if it was an acceptable distance away, the coordinates were used, if not they were generated again. Then, each asteroid was given a random scaling value in each direction to make them all different shapes.

Additionally, this project also includes several different light sources, including 1 point light and 6 spot lights. The ambient light is placed inside of the sun, which is rendered as being transparent to allow the light to pass through it. The 6 spot lights are used to light up the sun, since the ambient light is inside the sun, it is not lit up, hence needing a separate light source.
There are also 2 different rasterizer states, solid and wireframe. Through using the wireframe mode, you can better see that Venus has two models. This is because there is a second object to represent Venus' atmosphere that is slightly bigger than the planet and transparent.

The code for this project will be available in January 2022 after the assignment has been marked.
